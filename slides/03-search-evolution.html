Mathematical Models

Markov Decision Process

User search is a sequential process of actions (querying, clicking, etc.) to satisfy an information need. This can be modeled using a Markov Decision Process, where:

States are the current situation (e.g., viewing results).

Actions are available choices (e.g., click, query, stop).

Transitions are probabilities of moving between states: $P(S_{t+1} | S_t, A_t)$.

Abstraction of transition probabilities between user search actions. Baskaya et al. (2013).1

1Feza Baskaya et al. “Modeling behavioral factors ininteractive information retrieval”. CIKM 2013.

Mathematical Models

The Challenge with Basic Markov Models

A simple, first-order Markov model assumes the next action $S_{k}$ depends only on the immediate past action $S_{k-1}$:

$P(S_k | S_{k-1}, S_{k-2}, \dots) = P(S_k | S_{k-1})$

However, real user behavior is more complex. The probability of an action can depend on:

  • Which query is currently being examined? (e.g., Query 1 vs. Query 5)
  • What is the user's overall goal? (e.g., Exploratory vs. Known-item)

Our Goal: Enhance the basic Markov model by incorporating this missing context.

Mathematical Models

Extending the Markov Model

The Markov assumption states the future depends only on the present state. But we can define "state" however we like. We can enrich the state by bundling more historical context into it—a technique called state augmentation.

For example, we can define the state \(s_t\) as a bundle:

\(s_t = (q_t, a_{t-1}, q_0)\)

\(q_t\): The current query

\(a_{t-1}\): The previous action

\(q_0\): The session's initial query

By encoding history into the state vector, the model remains Markovian but becomes much more powerful and context-aware.

For capturing the entire history, one can use a Partially Observable MDP (POMDP), often with an RNN or Transformer.

Mathematical Models

Conditional Markov Model

Insight: The relevance of an action depends on which query it belongs to.

We make transitions conditional on both the previous action $Y_{k-1}$ and the query index $X_{k-1}$.

\( \underbrace{P\!\left(A_{j+1}=a^\star \mid Q_i,\;A_j\right)}_{\text{next-action probability}} \;=\; \frac{ \underbrace{\#\bigl(Q_i,\,A_j,\,a^\star\bigr)}_{\text{count of the (query, action, next-action) triplet}} }{ \underbrace{\#\bigl(Q_i,\,A_j\bigr)}_{\text{count of the (query, action) context}} } \)

Mathematical Models

Contextual Markov Model

Insight: Different users have different search intents or behaviors. We can't model them all with one model.

We split the data by user context and build a separate Markov model for each group.

Exploratory Search

  • Users learn about a topic
  • Longer sessions
  • More query reformulations
  • Exhaustive result list examination

Known-Item Search

  • Users look for a specific item
  • Shorter sessions
  • Quick query changes
  • Investigate only the top results

Mathematical Models

Query-change Markov Model

Insight: The way users syntactically modify their queries reveals their learning process.

We decompose the query change $\Delta q_i = q_i - q_{i-1}$ into:

  • $\Delta q_i^{=}$: Theme terms (kept from $q_{i-1}$)
  • $\Delta q_i^{+}$: Added terms (new in $q_i$)
  • $\Delta q_i^{-}$: Removed terms (deleted from $q_{i-1}$)

The Markov model states become these modification actions (keep, add, remove), capturing the user's thought process.

Cognitive Models

Typical user models see what users do (query, click), but often miss why they do it.

Real user behavior is driven by an evolving internal state of knowledge and intent. This is especially true in complex domains like academic digital libraries.

Our Idea: Explicitly model the user's Cognitive State to create more realistic simulations.

Introducing the Cognitive-Aware Complex Searcher Model (CACSM).

Cognitive Models

CACSM Framework

CACSM models the iterative cycle of a search session, with new decision points based on cognitive state.

An overview of the CACSM process flow. Orange elements are new cognitive components.

Cognitive Models

CACSM Components: Knowledge Sources

The model's "memory" is composed of four distinct knowledge types:

K1: Background Knowledge

Long-term, pre-existing domain understanding.

K3: Keyword Matching

Knowledge of direct term occurrences (syntactic).

K2: Knowledge Learnt

Short-term knowledge gained during the current session.

K4: Linguistic Knowledge

Semantic understanding of terms (synonyms, relations).

Cognitive Models

CACSM Components: Information Need (IN)

The model's "intent" is represented by a dynamic Information Need:

Fixed IN

The user has a specific, clear goal. (e.g., finding a known paper).

Exploring IN

The user is learning or Browse, with a flexible, open-ended goal.

CACSM uses a mixture of both to represent how a user's intent can shift during a single session.

Cognitive Models

CACSM Framework: The Update Mechanism

After each user action (e.g., a click), CACSM updates its cognitive state to simulate learning.

Example: Updating Knowledge Learnt (K2)

\[ \underbrace{P_{\text{new}}(K_2)}_{\substack{\text{Updated Short-Term} \\ \text{Knowledge}}} = \underbrace{P_{\text{old}}(K_2)}_{\substack{\text{Existing Short-Term} \\ \text{Knowledge}}} + \underbrace{\beta_{K2}}_{\substack{\text{Learning} \\ \text{Rate}}} \times \underbrace{P_{\text{interaction}}(K_2)}_{\substack{\text{Knowledge from} \\ \text{Recent Interaction}}} \]

\[ P_{\text{interaction}}(K_2) = \frac{\text{Impact of interaction for a specific outcome}}{\text{Total impact of all interactions}} \text{ ; } \text{Impact}(\text{doc}) = w_{\text{click}} \times \frac{1}{\text{rank}} \]

The term $P_{\text{interaction}}$ is weighted by functions that consider document rank and click status. A click on a low-ranked document signals a stronger learning moment.

Cognitive Models

CACSM Implementation

We implement the abstract CACSM framework using two concrete deep learning architectures:

CACSM$_{\texttt{UM}}$ (RNN-based)

A Gated Recurrent Unit (GRU) processes the sequence of user interactions to build a state representation.

Enhanced with negative sampling to better distinguish probable from improbable actions.

CACSM$_{\texttt{LLM}}$ (LLM-based)

The user's cognitive state is encoded and injected into a T5 language model via cross-attention.

This allows the LLM to generate predictions conditioned on both the textual history and the explicit cognitive state.

Summary

  • User search behavior can be modeled as a sequential decision process using Markov Models.
  • Simulations become more realistic by incorporating context (e.g., query changes, search type) or by using state augmentation for context-aware generation.
  • We introduced CACSM, a cognitive-aware framework that simulates a user's evolving internal state.
  • CACSM demonstrates that explicitly modeling a user's dynamic Knowledge Sources and Information Need significantly outperforms non-cognitive baselines.
